Performing High-Level Queries

The High-Level Queries panel offers the user a simple interface to perform some predefined complex queries, setting SQL conditions upon multiples database tables at once.




The left part of the window presents the list of all supported high-level queries; in the figure there are 2 available HLQs, respectively looking for all AOs with a given name and initial time greater than a given t0, and AOs with a given name and included among two given times. After the selection of the desired query on the list, the button 'Set' will allow to adjust the proper settings.

All the settings associated to the selected HLQ are displayed in the right part of the panel; in the example of the figure they are the name the queried AOs must match, and the upper and lower limit times of its time-series data (meaning the object must contain tsdata comprised among these given times).

Executing the query (click the 'Execute query' button) produces the same results as the 'Query' panel shown before.
